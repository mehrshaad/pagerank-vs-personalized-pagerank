PageRank vs Personalized PageRank

Both rank nodes by imagining a random surfer wandering the links. Most of the time the surfer follows an out-edge; every so often it gets bored and teleports. The one and only difference between the two algorithms: where the surfer teleports back to. Restart anywhere and you measure global importance. Restart at one node and you measure importance relative to that node. Restart in a set and you get Topic-Sensitive PageRank. Everything below is computed live — nothing is faked.

03 · The math

Same equation, one term changes

All three scores satisfy the same fixed-point equation. The link-following term on the right is identical everywhere — only the highlighted teleport term differs.

PageRank π(i) = (1−d) · 1/N + d · Σj→i π(j) / deg⁺(j)
Personalized πs(i) = (1−d) · es(i) + d · Σj→i πs(j) / deg⁺(j)
Topic-set πS(i) = (1−d) · 𝟙[i∈S]/|S| + d · Σj→i πS(j) / deg⁺(j)

Here d is the damping factor (typically 0.85), deg⁺(j) the out-degree of j, and es the indicator vector of the source s. PageRank spreads restart mass uniformly, so it measures global importance. Personalized PageRank puts all restart mass on s, so it measures importance relative to s — scores decay with distance from the source.

πv  =  (1−d) · v · (I − d·P)−1   ← closed form (row-vector convention, P row-stochastic)

Because the closed form is linear in the teleport vector v, personalization blends: παu+βw = α·πu + β·πw. This single fact is why Topic-Sensitive PageRank works — precompute a handful of basis vectors offline, then mix them per query at almost zero cost.

05 · Algorithms

Three ways to compute it

Every variant on this page runs the same algorithms — only the teleport vector e changes. The third one is the reason personalized PageRank scales to billion-node graphs.

# e — teleport vector: uniform (PR) · one-hot (PPR) · set indicator (TSPR)
# A_hat — row-normalized adjacency matrix
p = e.copy()
for t in range(T):
    p = (1 - d) * e  +  d * (A_hat.T @ p)
return p

Exact and simple. Each iteration touches every edge once — O(|E|) per step — and the error shrinks by ×d per step, so ~⌈1/log₁₀(1/d)⌉ iterations buy one extra digit. This is what the chart above runs.

visits = zeros(N)
v = sample(e)
for t in range(T):
    if rand() < 1 - d:  v = sample(e)              # bored → restart
    else:               v = random_out_neighbor(v)  # follow a link
    visits[v] += 1
return visits / visits.sum()                      # → π as T grows

Embarrassingly parallel and streaming-friendly. This is literally what the gold surfer does — the tick marks in the rank panel are this estimator running in your browser. Error decays like 1/√T, so it's great for rough top-k answers on huge graphs.

# Andersen–Chung–Lang 2006 — approximate PPR without touching the whole graph
p, r = zeros(N), e.copy()          # p: estimate · r: residual mass
while exists u with r[u] > eps * outdeg(u):
    p[u] += (1 - d) * r[u]
    for w in out(u):
        r[w] += d * r[u] / outdeg(u)
    r[u] = 0
return p

Local: the killer feature. Mass starts on the source and only pushes outward while it's still significant, so the run time depends on 1/eps — not on graph size. Sweeping the result by degree-normalized score also yields provably good local community cuts, which is why PPR is the workhorse of local clustering.

06 · The family

Beyond the pair — related approaches

Personalized PageRank sits inside a whole family of random-walk and mutual-recursion scores. Most differ only in what restarts, what decays, or what walks.

Haveliwala · 2002

Topic-Sensitive PageRank

Teleport into a set of pages per topic (sports, health, …), precompute one vector per topic offline, then blend at query time using the linearity of PPR. Personalization at web scale without a per-user computation.

Data mining · bioinformatics

Random Walk with Restart

The same mathematics as PPR under a different name. The RWR framing dominates in bioinformatics and data mining — prioritizing disease genes on protein networks, image captioning, cross-modal retrieval. If you can read this page, you can read RWR papers.

Kleinberg · 1999

HITS — hubs & authorities

No teleport at all. Two mutually recursive scores on a query-specific subgraph: good hubs point to good authorities and vice versa. Query-dependent like PPR, but the recursion is bidirectional and it's famously more sensitive to spam and topic drift.

Jeh & Widom · 2002

SimRank

“Two nodes are similar if their neighbors are similar.” Equivalent to asking how soon two random walkers starting at each node meet. Gives a pairwise similarity rather than a per-node importance — the natural sibling of PPR for “how alike are u and v?”.

Chung · 2007

Heat-Kernel PageRank

Swap PPR's geometric decay over walk lengths for a Poisson (e−ttᵏ/k!) weighting — a diffusion of heat from the seed. Sharper drop-off with distance, which often traces tighter community boundaries in local clustering.

GNNs · 2019 →

PPR inside neural networks

APPNP and PPRGo replace stacked message-passing layers with PPR propagation of learned features — decoupling “how far information travels” from network depth, and dodging the over-smoothing that plagues deep GNNs. The 1998 idea, load-bearing in 2020s architectures.

07 · In the wild

Where personalized PageRank earns its keep

RECOMMENDERS

“People / pins / repos like this”

Anchor the walk on a user (or the items they touched) and the top-scoring unvisited nodes are natural recommendations. Twitter's who-to-follow and Pinterest's related-pins systems were built on exactly this restart-at-the-user walk, computed with Monte Carlo at feed speed.

LOCAL CLUSTERING

Find the community around a seed

Run forward push from a seed, sort nodes by degree-normalized PPR, and sweep for the best cut. You recover the seed's community without ever reading the rest of the graph — with quality guarantees from the Andersen–Chung–Lang analysis.

GRAPH LEARNING

Propagation layers in GNNs

APPNP-style models predict features with a small MLP, then diffuse the predictions with personalized PageRank. Long-range structure at fixed parameter count — and PPRGo precomputes sparse PPR vectors to make it fast enough for web-scale graphs.

KNOWLEDGE GRAPHS · RAG

Entity-anchored retrieval

In knowledge-graph question answering and graph-flavored RAG, seed the walk on the entities mentioned in the query; the high-PPR neighborhood is the sub-graph worth feeding to the language model. Relevance-with-distance-decay is precisely what retrieval wants.
